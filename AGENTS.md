# OpenGravity Workspace Instructions

- Keep all code, comments, commit messages, and user-facing copy in English.
- Prefer small, reviewable changes over broad rewrites.
- Preserve BYOK and provider-agnostic behavior. Do not hardcode vendor assumptions.
- In Ask mode, answer only. In Planning mode, plan only. In Agent mode, keep actions explicit and reviewable.
- Prefer exact-match edits, scoped commands, and visible verification steps.
- Update tests when runtime behavior changes.
- Keep desktop UX focused. Do not surface every advanced control at once when a simpler default is possible.
